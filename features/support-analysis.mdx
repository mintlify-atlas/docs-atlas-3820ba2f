The Support Analysis page (`/soporte-analisis`) provides the results and analytics dashboard for the pressure vessel support system. After configuring design parameters in the [Support Design](/features/support-design) page, engineers use this view to review performance metrics, visualize load distributions, and examine activity data.

## Dashboard Overview

The page renders a 12-column responsive grid that organizes analytics widgets into logical groups. All widgets update based on the current design state.

### Metrics Row

At the top of the page, the **AnalyticsMetrics** component displays key performance indicators as a set of summary cards. These give an at-a-glance view of the most critical calculated values — such as total operating weight, base shear, overturning moment, and utilization ratios for the selected support type.

### Bar Chart

The **AnalyticsBarChart** component renders a full-width bar chart beneath the metrics row. This chart compares load contributions or calculated capacities across multiple categories, making it easy to identify which load case or component governs the design.

### Channel and Pages Breakdown

The center section contains two side-by-side widgets:

- **TopChannel** — ranks load sources or calculation paths by their relative contribution to the governing result
- **TopPages** — lists the most-used calculation routes or parameter combinations in the current session

### Active Users Chart

The **ActiveUsersChart** widget occupies the right side of the center row. In the context of support analysis, this widget visualizes time-series activity — for example, the history of design iterations or parameter changes during the current session.

### Acquisition and Session Charts

The lower section of the dashboard includes two additional time-series visualizations:

- **AcquisitionChannelChart** — shows how different load input channels (dead load, wind, seismic, thermal) contribute to the total demand over a parameter sweep
- **SessionChart** — tracks the progression of analysis results across multiple calculation runs

### Demographic Card

The **DemographicCard** widget provides a geographic or categorical breakdown. In the engineering context this may represent the distribution of load cases by code zone, site class, or vessel type across a project portfolio.

### Recent Activity Table

At the bottom of the page, the **RecentOrderAnalytics** table lists recent design actions or calculation events in a tabular format with timestamps and status indicators. This provides an audit trail of parameter changes and result updates.

## Workflow Integration

The Support Analysis page is the second step in the pressure vessel support workflow:

1. Define vessel and support parameters on the [Support Design](/features/support-design) page
2. Review computed results and visual analytics here on the Support Analysis page
3. Export the formal calculation report using the print button on the design page

<Note>
  The analytics dashboard reflects the parameters configured in the Support Design page. Navigate to `/soporte-diseno` first to enter your vessel geometry, loads, and support type before reviewing results here.
</Note>
